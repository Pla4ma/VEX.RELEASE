# PLAN: De-minify & Split VEX Codebase to ≤200 Lines Per File (FINAL — 2026-05-27)

## CONTEXT

VEX React Native app (Expo SDK 56, TypeScript 6.0.3 strict mode). `AGENTS.md` mandates no file exceeds 200 lines. **349 non-exempt source files exceed 200 lines** (excluding tests, 5,616-line auto-generated `supabase.ts`, and 55 deliberate barrel/auto-generated validation files).

**Already done:** Prettier de-minification ✓, FocusIdentityEngine split (955→128) ✓, ProfileScreen split (861→147) ✓, Settings service split (847→169) ✓, CoachRecommendationService split (877→123) ✓, validation directory reorganized into barrel architecture ✓.

**3 duplicate files still exist** from a failed script. **1 pre-existing tsc error** in `src/accessibility/AccessibilityEnhancer.ts` line 109.

**Codebase stats:** 1661 files in features/, 258 in session/, 321 in screens/, 102 in shared/. 54 feature directories.

## ABSOLUTE RULES FOR THE AI AGENT

1. **Typecheck after every file split**: `npx tsc --noEmit`. Fix errors before next file.
2. **No scripts for extraction.** Manual copy/paste. Scripts cause cascading import failures.
3. **Export everything.** Run `git grep "from.*OldFileName"` before extracting. Every function/type/const imported elsewhere must have `export`.
4. **Never change function signatures.** Move code only — no refactoring, no renaming.
5. **Keep original as re-export barrel** where imports depend on it. `git grep "from.*OldFileName"`.
6. **Run `npx tsc --noEmit` every 3-5 files.**
7. **Commit after every phase.**
8. **Don't move `this`-referencing methods** out of class files.
9. **Don't break React hooks order.** useState/useEffect deps must stay in same component.
10. **Don't forget `as const`** on config objects when moving.
11. **Named exports only** — barrel re-exports require named exports.
12. **No circular imports.** Extract shared deps to third file.
13. **Check barrel index.ts** for every feature — update re-exports after every split.
14. **Check test files** for direct imports from split source files.
